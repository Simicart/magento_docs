    Introduction Testing

    Welcome to the MAGENTO API from SIMICART TEAM! You can use our API to access MAGENTO API endpoints, which can get information on various products, customers, sales, and so on.

    Resource’s URI

    http(s)://{your_base_url}/simiconnector/rest/version/resources/{id}/nested_resources/{nested_id}?refines

    • version is v2
    • resources is the name of document that you want to work with. For example: products, customers, orders…
    • nested_resources is the name of document that related with the main resource (above). For example nested resources of products: images, categories, stock_item …

    Authentication

    To authorize, use this code:

    curl -X POST "api_endpoint_here" \
  -H "Content-Type:application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{}'

    Make sure to replace token with your OAuth2.0 API token.

    LavaHub uses OAuth2.0 to allow access to the API. You can register your developer account and your applications at our developer portal

    LavaHub expects for the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: Bearer <token>

    You must replace token with your OAuth2.0 API token.

    HTTP Verbs

    We use HTTP Verbs to determine the action with API resource. There are the table of HTTP Verbs are using for REST API:

    Verb Action Example
    POST Create POST /products
    Create new product

    POST /products/123
    Create new product with data similar as product 123
    GET Read GET /products
    Receive the list of products

    GET /products/123
    Receive the detail of product 123
    PUT Update PUT /products
    Mass update products

    PUT /products/123
    Update product 123
    DELETE Delete DELETE /products
    Mass delete warehouses with ids from query param

    DELETE /products/123
    Delete product 123

    Request

    All requests to MAGENTO API Gateway need an Authorization.

    Query params

    • To filter the result when listing the resource document, we use filter param. For example: products?filter[name][like]=John&filter[created_at]=2015-03-20T13:59:00Z&filter[status]=1

    • To get only some fields of document in the result, we use fields param. For example: products/123?fields=name,status

    Payload Body

    curl -X POST "https://abc.com/simiconnector/rest/v2/customers" \
  -H "Content-Type:application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
  "name": "John Doe",
  "email": "user@sample.com"
}'

    Pagination and sort order

    Pagination

    Request that presents multiple items will be paginated to 10 items by default. To control the pagination, we use three query params:

    Param Description Example
    limit Limit the number of items per page. Default value is 10. /products?limit=20
    Limit 20 products on result
    offset The number of items the be skipped on result. Default value is 0. /products?offset=20
    Get 10 products from 21th product
    page The page number of result. Default value is 1. If this param is presented, we ignore the offset param. /products?page=2
    Get 10 product of page 2 (that mean we skip 10 items)
    ids List id object want get in one query /products?ids=16,17
    Get 2 products has id = 16 and 17

    Sort order

    By the default, we will sort the result by _id field with desc direction. To control the sort order, we use two query params:

    Param Description Example
    order The attribute (field) that will be use for sorting. Default is _id /products?order=name
    We sort the result by name
    dir The sort direction. We accept two types asc and desc. Default is desc. /products?dir=asc
    We sort the result by increment _id

    Errors

    The error returns with JSON body like this:

    {
  "errors": [
    {
      "code": "1",
      "message": "Unsupported request method"
    }
  ]
}

    The LavaHub API uses the following HTTP error codes:

    Error Code Meaning
    400 Bad Request – Your request sucks
    401 Unauthorized – Your API key is wrong
    403 Forbidden – The resource requested is hidden from your permission
    404 Not Found – The specified resource could not be found
    405 Method Not Allowed – You tried to access a resource with an invalid method
    406 Not Acceptable – You requested a format that isn’t json
    410 Gone – The resource requested has been removed from our servers
    418 I’m a teapot
    429 Too Many Requests – You’re requesting too many resources! Slow down!
    500 Internal Server Error – We had a problem with our server. Try again later.
    503 Service Unavailable – We’re temporarially offline for maintanance. Please try again later.
